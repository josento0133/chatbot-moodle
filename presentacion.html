Funcionalidades

¿Qué puede hacer el asistente?

💬

Chat de dudas

El alumno escribe su pregunta y recibe una respuesta adaptada a Bachillerato en segundos. El historial persiste entre sesiones (últimos 10 mensajes de contexto).

📝

Ejercicios prácticos

Para Matemáticas, Física, Química y Biología, el asistente genera un ejercicio con pregunta estructurada. El alumno responde y recibe corrección + explicación.

📊

Seguimiento de progreso

Panel con: mensajes enviados hoy, total de ejercicios, % de acierto, y desglose por materia (temas más consultados).

🛡️

Límite diario

Máximo 50 mensajes por usuario al día (configurable). Evita abuso y garantiza recursos disponibles para todos los alumnos.

Base de datos

Tres tablas en Moodle

Tabla Campos principales Propósito
block_chatbot_messages user_id · role · content · subject · timestamp Historial de conversación por usuario
block_chatbot_exercises user_id · question · answer · correct · timestamp Ejercicios generados y respuestas del alumno
block_chatbot_progress user_id · subject · topic · count · last_seen Temas más consultados por materia

Las tablas se instalan automáticamente vía install.xml (XMLDB) al añadir el plugin. El motor de base de datos lo hereda Moodle: SQLite en dev, MySQL/MariaDB en prod.

Seguridad

Autenticación JWT HS256

// PHP — Moodle genera el token en el servidor private function generate_jwt(int $user_id, string $secret): string { $header = $this->b64url(json_encode(['alg'=>'HS256', 'typ'=>'JWT'])); $payload = $this->b64url(json_encode(['user_id'=>$user_id, 'exp'=>time()+3600])); $sig = $this->b64url(hash_hmac('sha256', "$header.$payload", $secret, true)); return "$header.$payload.$sig"; } # Python — FastAPI verifica el token con PyJWT def verify_jwt(token: str, secret: str = None) -> dict: return jwt.decode(token, secret, algorithms=["HS256"])

Secreto compartido

Configurado en Moodle admin panel. El mismo secreto en PHP y Python garantiza que solo Moodle puede generar tokens válidos.

Expiración 1 hora

El campo exp en el payload invalida automáticamente el token tras 60 minutos de inactividad.

user_id en payload

Cada endpoint verifica que el user_id del token coincide con el recurso solicitado. Imposible leer el historial de otro alumno.

Despliegue

Poner en marcha el sistema

1

Levantar Moodle + MySQL con Docker

docker compose up -d — arranca Moodle 4 en localhost:8080 con MySQL 8

2

Instalar el plugin en Moodle

Copiar moodle-plugin/block_chatbot/ a moodle/blocks/ → admin → notificaciones → instalar

3

Configurar en admin de Moodle

Plugins → Bloques → Chatbot: fijar backend_url y jwt_secret

4

Arrancar el backend FastAPI

cd backend && uvicorn main:app --reload --port 8000

5

Lanzar el modelo LLM

ollama run llama3 — descarga y sirve el modelo en localhost:11434

6

Añadir el bloque a un curso y probar

Modo edición → "Añadir bloque" → Chatbot Académico → ya funciona para todos los alumnos del curso
